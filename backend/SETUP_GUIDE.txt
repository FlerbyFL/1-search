## STEP 1: Установка зависимостей

### Windows:
1. Установите PostgreSQL: https://www.postgresql.org/download/windows/
2. Установите Go: https://golang.org/dl/
3. Добавьте оба в PATH

### macOS:
```bash
brew install postgresql go
```

### Linux (Ubuntu/Debian):
```bash
sudo apt-get install postgresql postgresql-contrib golang-go
```

## STEP 2: Создание базы данных

### Способ 1: Автоматически (при первом запуске приложение создаст таблицы)
Просто запустите приложение, оно создаст все таблицы автоматически.

### Способ 2: Вручную
```bash
# Подключитесь к PostgreSQL
psql -U postgres

# Создайте БД
CREATE DATABASE e_catalog;

# Выйдите
\q
```

## STEP 6: Docker Deployment (рекомендуется для production)

### Запуск всех контейнеров:
```bash
# 1. Создайте .env в корне проекта
cp backend/.env.example .env

# 2. Отредактируйте переменные если нужно
# vim .env

# 3. Запустите контейнеры
docker-compose up -d

# 4. Проверьте статус
docker-compose ps

# 5. Просмотрите логи
docker-compose logs -f go-backend
```

### Остановка:
```bash
docker-compose down
```

### Удаление данных:
```bash
docker-compose down -v
```

## STEP 9: Планирование регулярного парсинга (опционально)

### Использование cron jobs (Linux/Mac):
```bash
# Отредактируйте crontab
crontab -e

# Добавьте строку для парсинга каждый день в 3 AM
0 3 * * * curl -H "X-Parse-Token: your_token" http://localhost:8080/api/parse-all
```

### Использование Windows Task Scheduler:
1. Откройте Task Scheduler
2. Create Basic Task
3. Set trigger: Daily at 03:00 AM
4. Action: Start a program
5. Program: `curl.exe`
6. Arguments: `-H "X-Parse-Token: your_token" http://localhost:8080/api/parse-all`

### Использование Go cron (в коде):
```go
import "github.com/robfig/cron/v3"

func main() {
    c := cron.New()
    c.AddFunc("0 3 * * *", func() {
        // Вызовите парсинг
        scrapeAllStoresAndSave()
    })
    c.Start()
}
```

## TROUBLESHOOTING

### Проблема: "Error: parse-all endpoint failed"
- Проверьте что PostgreSQL запущена
- Проверьте переменные окружения в .env
- Убедитесь что БД создана

### Проблема: "Connection refused"
- Убедитесь что приложение на порту 8080
- Проверьте firewall
- Проверьте что приложение запущено

### Проблема: "Database locked"
- Остановите все соединения к БД
- Перезагрузите PostgreSQL
- Очистите кэш приложения

### Проблема: "Parsing timeout"
- Увеличьте таймаут в parser.go (сейчас 60 сек)
- Проверьте скорость интернета
- Проверьте доступность магазинов

## PERFORMANCE TIPS

1. **Используйте индексы:**
   ```sql
   SELECT * FROM pg_stat_user_indexes;
   ```

2. **Оптимизируйте запросы:**
   ```sql
   EXPLAIN ANALYZE SELECT * FROM products WHERE product_name ILIKE '%%';
   ```

3. **Мониторьте размер БД:**
   ```sql
   SELECT pg_size_pretty(pg_database_size('e_catalog'));
   ```

4. **Очищайте старые данные:**
   ```bash
   curl -H "X-Parse-Token: token" http://localhost:8080/api/cleanup-old
   ```

## BACKUP & RESTORE

### Создание резервной копии:
```bash
pg_dump -h localhost -U postgres e_catalog > backup_$(date +%Y%m%d).sql
```

### Восстановление из резервной копии:
```bash
psql -h localhost -U postgres e_catalog < backup_20240226.sql
```
